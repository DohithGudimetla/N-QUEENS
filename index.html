01 — The Problem

A Classic Puzzle
of Constraint Satisfaction

The N-Queens problem asks: can you place N chess queens on an N×N chessboard so that no two queens threaten each other? Queens can attack horizontally, vertically, and diagonally.

Combinatorial Explosion

For an 8×8 board, there are 4,426,165,368 ways to place 8 queens. Only 92 are valid solutions.

Historical Significance

First proposed in 1848 by chess composer Max Bezzel, this problem became foundational in computer science and AI.

Three Constraints

No two queens may share the same row, column, or diagonal. This creates a deeply interconnected constraint network.

02 — The Algorithm

Understanding
Backtracking

The algorithm explores possibilities and retreats when conflicts appear. It places queens one row at a time, checking constraints at each step.

01

Place Queen

Try placing a queen in the first available column of the current row.

02

Check Conflicts

Verify if the placed queen conflicts with any existing queens along rows, columns, or diagonals.

03

Backtrack

If no valid position exists in the current row, remove the last queen and try the next column in the previous row.

04

Solution Found

When all N queens are placed without conflicts, a valid solution is recorded. Continue searching for more.

backtrack.pseudo 
function solveNQueens(board, row, N):
    if row == N:
        // All queens placed — solution found!
        return true
    
    for col in 0 .. N-1:
        if isSafe(board, row, col):
            board[row] = col      // Place queen
            if solveNQueens(board, row + 1, N):
                return true
            board[row] = -1      // Backtrack
    
    return false           // No valid placement
